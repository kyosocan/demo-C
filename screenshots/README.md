# 截图目录说明

本目录用于存放需求文档中所有功能的截图。

## 截图文件命名规则

截图文件命名格式：`需求编号-功能名称.png`

例如：
- `REQ-UI-001-广场页面.png`
- `REQ-CM-001-资料详情页.png`
- `REQ-SF-001-搜索功能.png`

## 截图要求

1. **尺寸要求**：
   - 移动端截图宽度：480px
   - 高度：根据实际内容，建议保持16:9或9:16比例

2. **质量要求**：
   - 分辨率：至少2倍图（960px宽度）或更高
   - 格式：PNG格式，支持透明背景
   - 清晰度：文字和图标清晰可读

3. **内容要求**：
   - 截图应展示功能的主要界面
   - 包含关键交互元素
   - 展示典型的数据内容
   - 如有必要，可添加标注说明

4. **截图工具建议**：
   - 浏览器开发者工具（Chrome DevTools）
   - 移动端模拟器
   - 截图工具（如Snipaste、FastStone Capture等）

## 如何添加截图

1. 运行应用：`npm run dev`
2. 在浏览器中打开应用（建议使用移动端视图）
3. 导航到对应功能页面
4. 使用截图工具截取界面
5. 按照命名规则保存到本目录
6. 在需求文档中，截图会自动显示
